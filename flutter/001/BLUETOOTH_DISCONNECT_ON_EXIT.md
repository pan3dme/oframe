# 蓝牙页面退出时自动断开连接功能

## 功能说明

当用户从蓝牙页面退出(返回或切换到其他页面)时,系统会自动执行以下操作:

1. **如果正在同步** → 先发送停止同步命令给设备
2. **如果已连接蓝牙** → 主动断开蓝牙连接
3. **如果正在上传** → 取消上传任务
4. **清理资源** → 取消所有订阅和扫描

## 实现原理

### dispose()方法增强

在`bluetooth_page.dart`的`dispose()`方法中添加了智能清理逻辑:

```dart
@override
void dispose() {
  // 1. 如果正在同步，先停止同步
  if (_isSyncing) {
    print('[蓝牙] 页面退出，正在停止同步...');
    _stopSync();  // 发送syncing=false命令
  }
  
  // 2. 如果已连接，主动断开蓝牙连接
  if (_isConnected && _connectedDevice != null) {
    print('[蓝牙] 页面退出，正在断开蓝牙连接...');
    _disconnect();  // 断开蓝牙连接
  }
  
  // 3. 取消上传
  if (_isUploading) {
    _isUploading = false;
    print('[蓝牙] 页面退出，取消上传');
  }
  
  // 4. 取消所有订阅
  _connectionStateSubscription?.cancel();
  _scanResultSubscription?.cancel();
  _notifySubscription?.cancel();
  
  // 5. 停止扫描
  FlutterBluePlus.stopScan();
  
  super.dispose();
}
```

## 执行流程

### 场景1: 正在同步时退出

```
用户点击返回
   ↓
检测到 _isSyncing = true
   ↓
调用 _stopSync()
   ↓
发送 {syncing: false} 命令给设备
   ↓
取消通知订阅
   ↓
断开蓝牙连接
   ↓
清理所有资源
   ↓
页面销毁
```

### 场景2: 已连接但未同步时退出

```
用户点击返回
   ↓
检测到 _isConnected = true
   ↓
调用 _disconnect()
   ↓
断开蓝牙连接
   ↓
清理所有资源
   ↓
页面销毁
```

### 场景3: 未连接时退出

```
用户点击返回
   ↓
检测到未连接
   ↓
跳过同步和断开步骤
   ↓
直接清理资源
   ↓
页面销毁
```

## 优势

### 1. 防止资源泄漏
- ✅ 自动断开蓝牙连接,避免后台占用
- ✅ 取消所有Stream订阅,防止内存泄漏
- ✅ 停止扫描,节省电量

### 2. 优雅的设备交互
- ✅ 先通知设备停止同步,再断开连接
- ✅ 设备端能正确收到停止信号
- ✅ 避免设备端仍在发送数据但手机端已关闭

### 3. 用户体验优化
- ✅ 无需手动断开连接
- ✅ 下次进入页面可以重新扫描连接
- ✅ 避免"设备忙"或"连接超时"问题

## 注意事项

### 异步操作的局限性

由于`dispose()`是同步方法,而`_stopSync()`和`_disconnect()`是异步方法,存在以下情况:

1. **命令可能未完全发送** - 页面销毁太快,蓝牙命令可能还在队列中
2. **解决方案** - 这些方法内部已有错误处理,即使失败也不会崩溃

### 改进建议(可选)

如果需要确保命令一定发送成功,可以考虑:

```dart
// 方案1: 添加短暂延迟
await Future.delayed(Duration(milliseconds: 500));

// 方案2: 使用WillPopScope拦截返回
WillPopScope(
  onWillPop: () async {
    await _cleanupBeforeExit();
    return true;
  },
  child: Scaffold(...),
)
```

但当前实现已经足够,因为:
- 蓝牙命令发送很快(通常<100ms)
- 即使命令丢失,设备端也有超时机制
- 简单可靠,不会阻塞UI

