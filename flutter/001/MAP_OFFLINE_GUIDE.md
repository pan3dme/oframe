# 地图离线缓存功能说明

## 功能概述

地图中心页面现已支持**完全离线显示**,包括:
1. ✅ **地图瓦片(底图)离线缓存** - 自动缓存访问过的区域
2. ✅ **道路和地名数据离线缓存** - SQLite数据库存储
3. ✅ **设备位置数据离线缓存** - SQLite数据库存储

## 使用方法

### 1. 在线时预缓存地图区域

在有网络连接时:
- 打开地图中心页面
- 浏览你希望离线查看的地图区域
- 系统会**自动缓存**所有访问过的瓦片

### 2. 离线使用

断开网络后:
- 已缓存的地图区域仍可正常显示
- 已加载的道路、地名和设备位置仍可查看
- 可以平移、缩放查看已缓存区域

### 3. 管理缓存

点击右上角的**云下载图标**:
- 查看缓存状态
- 清除所有缓存释放空间

## 技术实现

### 地图瓦片缓存
- 使用 `flutter_map_tile_caching` v9.0.1
- 基于 ObjectBox 数据库存储
- 缓存位置:
  - iOS: Documents目录
  - Android: 应用数据目录
- 缓存长期有效,除非手动清除

### 业务数据缓存
- 使用 SQLite (sqflite)
- 存储内容:
  - 道路数据 (map_routes表)
  - 地名数据 (map_places表)  
  - 设备数据 (devices表)
  - 设备位置 (device_lot表)

## 注意事项

1. **首次使用需要网络** - 必须先在线访问才能缓存
2. **缓存不会自动更新** - 定期联网更新可获得最新数据
3. **缓存占用空间** - 大量缓存可能占用较多存储空间,可定期清理
4. **未缓存区域离线时无法显示** - 只显示之前访问过的区域

## 数据加载策略

采用**"先缓存后网络"**策略:
1. 页面加载时优先从本地缓存读取
2. 快速显示已有数据
3. 后台尝试从网络更新
4. 网络失败不影响已显示的缓存数据

这样即使网络不佳或离线,也能立即看到上次打开的内容。
